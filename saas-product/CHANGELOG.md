# CHANGELOG — saas-product

Версия шаблона = номер верхней записи. Растёт на +1 при изменении, которое стоит распространять в проекты, инстанцированные из этого шаблона; мелкие правки формулировок версию не двигают. Теги: `[base]` — из общего скелета base-template, `[saas]` — собственная специфика класса.

Выровнен с **base-template@v8**.

## v7 — 2026-06-18
`[saas][инструкция]` Рабочий слой `scripts/` — код-черновик/эксперименты: отдельная top-level папка между `src/` (продукт-код) и `output/` (документы-черновики). Ленивая — в дереве «Архитектура» и в bootstrap «Что НЕ создаётся» (появляется при первом скрипте, не пустой на старте). Источник — потребность боевого saas-проекта.

## v6 — 2026-06-18
`[base][инструкция]` Лог-дисциплина (`methodology/index-log-format.md` + `methodology/lint.md`): дата записи в `wiki/log.md` — дата окружения на момент записи, не mtime/дата работы; инвариант append-only «дата ≥ предыдущей»; lint в «только отчёт» ловит немонотонность, без автосортировки. Источник — рецидив mtime-датирования в боевом проекте. (Данные не трогаются.) Закладка прыгает @v5→@v8: base v6/v7 — про роли, у этого класса ролей нет (N/A).
