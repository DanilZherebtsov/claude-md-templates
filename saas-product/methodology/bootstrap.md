# Bootstrap — инициализация под конкретный продукт

Запускается при первом ingest в проект без структуры или по просьбе («инициализируй структуру», «разверни вики»). **Никогда** из `query`/`lint`: нет структуры — Claude просит сначала что-нибудь дать. Один раз при старте; повторно — только восстановление разрушенного.

Bootstrap — короткая настройка под продукт. Шаблон уже содержит `CLAUDE.md`, `STATE.md`, `methodology/`; bootstrap достраивает рабочие слои.

## Шаг 1 — короткое интервью (3–5 вопросов)

- Что за продукт и для кого; текущая стадия?
- Какие источники будут приходить (интервью/discovery, заметки решений, техдокументация)?
- Уже есть код (`src/`) или runtime-данные (`data/`)?

## Шаг 2 — предложить структуру

Claude предлагает раскладку `raw/` под названные источники — по умолчанию `discovery/`, `decisions/`, `technical/` (+ `business/`, `brainstorms/` на ранней стадии). Человек подтверждает/правит.

## Шаг 3 — создать (чего нет, не перезаписывая существующее)

- `raw/.gitkeep` + согласованные подпапки.
- `wiki/` + 5 type-папок (`decisions/`, `architecture/`, `discovery/`, `synthesis/`, `principles/`, каждая с `.gitkeep`); `wiki/index.md` (`# Wiki Index`); `wiki/log.md` (`# Wiki Log` + `## [YYYY-MM-DD] bootstrap | инициализация`).
- `output/.gitkeep` — корень пустой (`drafts/` — по факту); `specs/.gitkeep` (файлы спек `<feature>-NNN-...` — по факту, при взятии задачи в работу).

## Шаг 4 — заполнить сопутствующие файлы из ответов интервью

- «О проекте» в `CLAUDE.md` (что строим, для кого, ограничения). После заполнения **удалить инструктивную сноску** — блок `>` под заголовком «О проекте» (он для составителя, не для проекта).
- **Дерево «Архитектура» в `CLAUDE.md`** — отразить созданные подпапки `raw/`, если отличаются от стандартных.
- Стартовый раздел `STATE.md` (стадия); `_Обновлено:_` = сегодня.

## Что НЕ создаётся при bootstrap

- `src/`, `data/`, `docs/` — появляются по решению человека, когда нужны.
- Подпапки `raw/` сверх согласованных — по факту (правило «Структура растёт по мере надобности» в [CLAUDE.md](../CLAUDE.md)).

После bootstrap — если проект под git — один коммит `bootstrap: инициализация структуры`; без git шаг пропускается.
