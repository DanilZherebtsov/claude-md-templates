# Lint — проверка здоровья вики

Запускается фразами: «прогони lint», «проверь вики», «проверь целостность». Стартовый триггер свежести (когда сам предложить прогон) — в CLAUDE.md, §5 «Дисциплины».

Периодическая проверка. Два уровня с разной степенью полномочий.

## Автоисправление (детерминированное — Claude правит сам)

- Страницы, отсутствующие в `wiki/index.md` → добавить с заглушкой `(no summary)`.
- Записи индекса, указывающие на удалённые файлы → пометить `[MISSING]` (не удалять; решение оставить за человеком).
- Битые внутренние ссылки, для которых есть ровно один файл с тем же именем в другом месте → починить путь.
- Ссылки в поле `sources:`, указывающие на перемещённые сырые файлы с единственным кандидатом → починить.
- Отсутствующие взаимные ссылки между очевидно связанными страницами в одной папке → добавить.

## Только отчёт (эвристическое — требует суждения человека)

- Страницы-сироты без входящих ссылок.
- Концепции, упомянутые на нескольких страницах, но без собственной страницы.
- Устаревшие утверждения, противоречащие более новому источнику.
- ADR-ы со статусом `active`, чьё решение противоречит более новому ADR.
- Страницы больше 800 строк.
- Страницы без поля `sources:`.
- Страницы `type: architecture` без поля `implementation:` или с битыми путями в нём (code drift — код был переименован/удалён, страница ссылается в пустоту).
- **Нарушение глубины:** любой `.md` в `wiki/<type>/<subdir>/...` — страницы вики должны быть плоско в type-folder (см. правило про расширения схемы в «Дисциплине» в [CLAUDE.md](../CLAUDE.md)). Исключение — `wiki/index.md` и `wiki/log.md` на уровне `wiki/`.

## Проверка обновлений шаблона

Этот проект инстанцирован из версионируемого шаблона (имя и версия — в строке-метке вверху `CLAUDE.md`). На каждом прогоне lint:

- Прочитать апстрим-`CHANGELOG.md` своего шаблона: `https://raw.githubusercontent.com/DanilZherebtsov/claude-md-templates/main/<имя-шаблона>/CHANGELOG.md` (`<имя-шаблона>` — из строки-метки вверху `CLAUDE.md`).
- Сравнить верхнюю версию апстрима с версией этого проекта.
- Апстрим новее → кратко пересказать новые записи changelog (что и зачем) и **предложить** обновление. Применять только с подтверждения человека и точечно: проект мог разойтись со шаблоном в своих разделах — это смысловой перенос, не слепая замена.
- Версия совпадает или апстрим недоступен → ничего не выводить.

После lint — дописать в `wiki/log.md`:
```
## [YYYY-MM-DD] lint | найдено N проблем, исправлено M
```
