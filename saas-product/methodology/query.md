# Query — структурированный запрос с цитатами

Запускается фразами: «что мы знаем про X», «сделай справку по Y», «сравни A и B на основе вики», «обоснуй решение Z по вики».

Используется, когда нужно формальное ревью знания, а не просто помощь по задаче.

1. Прочитать `wiki/index.md` и отобрать страницы-кандидаты — фильтруя по **префиксу имени** и, при необходимости, `grep` по полю `tags:` во frontmatter. В контекст идут только релевантные страницы, не вся вика.
2. Прочитать эти страницы (и пройти по ссылкам на один шаг, если нужно). При вопросах вида «что мы знаем про…» — **сначала вики, потом сырые источники**, не по памяти разговора.
3. Синтезировать ответ с цитатами в формате markdown-ссылок: `[Название страницы](wiki/<type>/<slug>.md)` для wiki, `[Описание](raw/<концерн>/<file>.md)` для сырых файлов, `[Название](https://...)` для внешних URL. Уровень уверенности читается из структуры (число и независимость источников, связанные решения, наличие контр-аргументов), не из числовых оценок — ответ это отражает.
4. Если вопрос вскрыл пробел — предложить шаг дальше: новую страницу, недостающий источник, устаревшее утверждение, которое стоит перепроверить.
5. Если ответ содержательный (сравнение, ретроспектива, обоснование решения, брифинг) — предложить записать его обратно в `wiki/synthesis/`. **Записанные ответы накапливаются. Ответы в чате — испаряются.**

Обычные запросы не пишут в файлы. Только запись-обратно (после явного «да») и явный ingest изменяют вики.
