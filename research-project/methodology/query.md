# Query — структурированный запрос с цитатами

Запускается фразами: «что мы знаем про X», «какие свидетельства за/против Y», «сделай обзор по теме Z», «сравни позиции A и B», «обоснуй методологический выбор C по вики».

Используется, когда нужно формальное ревью знания, а не просто помощь по задаче.

1. Прочитать `wiki/index.md`, чтобы найти страницы-кандидаты.
2. Прочитать эти страницы (и пройти по ссылкам — `supports / contradicts / refines` для claim'ов, `supersedes / superseded_by` для decisions — на один шаг, если нужно).
3. Синтезировать ответ с цитатами в формате markdown-ссылок: `[Название страницы](wiki/<type>/<slug>.md)` для wiki, `[Описание](raw/<category>/<file>.md)` для сырых файлов, `[Название](https://...)` для внешних URL.
4. **Различать уровни уверенности по структуре вики, а не по числам:** claim со статусом `validated` и тремя независимыми Evidence — это сильное утверждение; claim со статусом `open` и одним источником — гипотеза; claim с непустым Counter-evidence — спорное место, ответ обязан это отразить.
5. Если вопрос вскрыл пробел — предложить шаг дальше: новый claim, недостающий источник, противоречие, требующее разрешения, устаревшее утверждение, которое стоит перепроверить по оригиналу.
6. Если ответ содержательный (обзор, сравнение, ретроспектива, обоснование методологического выбора, брифинг) — предложить записать его обратно в `wiki/synthesis/`. **Записанные ответы накапливаются. Ответы в чате — испаряются.**

Обычные запросы не пишут в файлы. Только запись-обратно (после явного «да») и явный ingest изменяют вики.

## Особые формы query

**«Какие claim'ы зависят от источника X?»** — обратный поиск: пройти по `sources:` всех claim'ов в `wiki/claims/`, найти ссылающиеся на X. Полезно, когда источник опровергнут или переинтерпретирован — нужно понять, какие выводы это затронет.

**«Дай evidence-карту по теме Y»** — найти все claim'ы по тегам или префиксу имени; построить таблицу: claim → статус → число supporting Evidence → число Counter-evidence → ключевые источники. Это полезный pre-write шаг перед написанием раздела deliverable.

**«Где у нас противоречия?»** — найти claim'ы с непустыми Counter-evidence или связями `contradicts`. Это inventory спорных мест проекта; полезно перед финальным синтезом.
