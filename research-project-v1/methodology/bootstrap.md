# Bootstrap — инициализация структуры проекта

Запускается, если в проекте ещё нет `wiki/`, `output/`, `STATE.md` или они частично разрушены.

Нужно один раз при инициализации проекта; повторно — только при восстановлении структуры (например, после случайного удаления).

## Что создаётся

Claude создаёт то, чего нет (никогда не перезаписывает существующее):

- `raw/.gitkeep`
- `wiki/.gitkeep`
- `output/.gitkeep`
- `wiki/index.md` с заголовком `# Wiki Index` (тело пустое; форматируется по [methodology/index-log-format.md](index-log-format.md)).
- `wiki/log.md` с заголовком `# Wiki Log` и первой записью `## [YYYY-MM-DD] bootstrap | инициализация`.
- Четыре type-папки внутри `wiki/`: `claims/`, `decisions/`, `synthesis/`, `principles/` (каждая с `.gitkeep`).
- `output/drafts/.gitkeep` (рабочие файлы вопросов `q-NNN-...` кладутся прямо в `output/` по факту, заранее не создаются).
- `STATE.md` в корне — стартовый шаблон с 7 пустыми разделами (структура — [methodology/state-rules.md](state-rules.md)). Поле `_Обновлено:_` ставится в сегодняшнюю дату.

## Что не создаётся автоматически

- `src/`, `data/` — появляются по решению человека, когда нужны (исследование с кодом анализа; исследование с собственными датасетами).
- Подпапки внутри `raw/` (`literature/`, `fieldwork/`, `conversations/`, `decisions/`, `datasets/` и т.п.) — заводятся по факту, когда появляется первый файл соответствующего типа.

## Триггер

Bootstrap запускается:
- **Автоматически** — при первом ingest в проект без структуры. Claude обнаруживает отсутствие `wiki/`, делает bootstrap, потом ingest.
- **По явной просьбе** — «инициализируй структуру», «развёрни вики».
- **Никогда не запускается из `query` или `lint`:** если структуры нет, Claude отвечает человеку — «сначала предоставь что-нибудь, чтобы я инициализировал вики».

После bootstrap — один коммит с сообщением `bootstrap: инициализация структуры`.
