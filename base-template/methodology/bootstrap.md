# Bootstrap — инициализация под конкретный проект

Запускается при первом ingest в проект без структуры или по просьбе («инициализируй структуру», «разверни вики»). **Никогда** из `query`/`lint`. Один раз; повторно — только восстановление.

Bootstrap — не штамповка фиксированных папок, а **короткая настройка под проект**. Шаблон уже содержит `CLAUDE.md`, `STATE.md`, `methodology/`, `roles/`; bootstrap достраивает рабочие слои.

## Шаг 1 — короткое интервью (3–5 вопросов)

- Что за проект и в чём сейчас цель?
- Какие источники будут приходить?
- Доменные конвенции, если не по умолчанию (валюта/единицы/формат цитат — слот S8).

## Шаг 2 — предложить структуру

На основе ответов Claude **предлагает** раскладку (не создаёт молча): подпапки `raw/` под названные источники + `прочее/`; особые папки, если всплыли. Человек подтверждает/правит.

## Шаг 3 — создать (чего нет, не перезаписывая существующее)

- `raw/.gitkeep` + согласованные подпапки.
- `wiki/` + type-папки класса (<<СЛОТ S2>>, каждая с `.gitkeep`); `wiki/index.md` (`# Wiki Index`); `wiki/log.md` (`# Wiki Log` + `## [YYYY-MM-DD] bootstrap | инициализация`).
- Рабочий слой класса — **корень создаётся пустым** (`.gitkeep`): <<СЛОТ S4: `output/` и/или `documents/`; для классов с кодом также `specs/`>>. Подпапки (`drafts/`, `черновики/`…) — по факту, не заранее.

## Шаг 4 — заполнить сопутствующие файлы

- «О проекте» в `CLAUDE.md` из ответов интервью.
- **Дерево «Архитектура» в `CLAUDE.md`** — вписать созданные подпапки `raw/` (в шаблоне их нет).
- Стартовый раздел `STATE.md` (профиль/стадия); `_Обновлено:_` = сегодня.

## Что НЕ создаётся при bootstrap

- **Роли** — заранее не заводим; сообщить, что любую роль можно создать позже «создай роль <название>» (см. [roles.md](roles.md)).
- Папки сверх минимума, **включая подпапки рабочего слоя** (`drafts/`, `черновики/`), — по смыслу, когда появится потребность (правило «Структура растёт по мере надобности» в [CLAUDE.md](../CLAUDE.md)).

После bootstrap — если под git — коммит `bootstrap: инициализация структуры`; без git пропускается.
