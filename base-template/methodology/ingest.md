# Ingest — добавление нового знания

Запускается фразами: «обработай этот документ», «запомни это», «зафиксируй», «добавь в вики».

Когда человек указывает на новый сырой источник или кладёт файл в `raw/`:

1. **Прочитать источник.** Длинный — читать по частям, не пробегать. Сначала обсудить ключевые выводы и подтвердить намерение перед изменением вики.
2. **Разместить сырой файл.** В подходящую подпапку `raw/<концерн>/YYYY-MM-DD-slug.md`; нет подходящей — завести новую и вписать в дерево `CLAUDE.md` (правило «Структура растёт по мере надобности»). Сохранить оригинал. **Удалить секреты** (пароли, ключи, лишние перс.данные). Добавить метаданный заголовок: откуда, дата получения, дата документа (если известна).
3. **Скомпилировать в вики.** Для каждой затронутой страницы:
   - **Тот же тезис** → объединить, добавить источник в `sources:`, обновить точечно.
   - **Новая концепция** → новая страница в подходящей type-папке, имя по концепции (не по имени файла). Для типов с префиксной группировкой — использовать префикс. Выбор типа и формат — [page-conventions.md](page-conventions.md).
   - **Противоречие** → аннотировать конфликт в тексте, обозначить обе стороны, вынести на разрешение человеком. Не выбирать молча.
4. **Каскад.** Просканировать ту же type-папку и индекс на материально затронутые страницы. Обновить, освежить `updated:`.
5. **Обновить `wiki/index.md`** — записи затронутых страниц. Формат — [index-log-format.md](index-log-format.md).
6. **Дописать в `wiki/log.md`:**
   ```
   ## [YYYY-MM-DD] ingest | <название основной страницы>
   - Updated: <каскадно-обновлённая страница>
   ```
7. **Закоммитить — если проект под git.** Один коммит, сообщение `ingest: <источник>`. Без git — шаг пропускается (история в `log.md` и датах страниц).

Важное правило: **при обновлении любого утверждения — перечитайте сырой источник, на который оно ссылается.** Не относитесь к тексту вики как к первоисточнику — так заводится дрейф.
