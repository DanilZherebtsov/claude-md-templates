# Bootstrap — инициализация под конкретное дело

Запускается при первом ingest в проект без структуры или по просьбе («инициализируй структуру», «разверни вики»). **Никогда** из `query`/`lint`: нет структуры — Claude просит сначала что-нибудь дать. Один раз при старте; повторно — только восстановление разрушенного.

Bootstrap — не штамповка фиксированных папок, а **короткая настройка под дело**. Шаблон уже содержит `CLAUDE.md`, `STATE.md`, `methodology/`, `roles/`; bootstrap достраивает рабочие слои под конкретный бизнес.

## Шаг 1 — короткое интервью

Claude задаёт 3–5 наводящих вопросов (не больше) и ждёт ответов:
- Что за дело и в чём сейчас цель?
- Какие источники будут приходить? (договоры, прайсы, переписка, ресёрч…)
- Основная валюта и формат дат, если не по умолчанию.

## Шаг 2 — предложить структуру

На основе ответов Claude **предлагает** раскладку (не создаёт молча): подпапки `raw/` под названные источники + `прочее/`; особые папки, если всплыли. Человек подтверждает или правит.

## Шаг 3 — создать (чего нет, не перезаписывая существующее)

- `raw/.gitkeep` + согласованные подпапки.
- `wiki/` + пять type-папок (`decisions/`, `entities/`, `discovery/`, `synthesis/`, `principles/`, каждая с `.gitkeep`); `wiki/index.md` (`# Wiki Index`); `wiki/log.md` (`# Wiki Log` + запись `## [YYYY-MM-DD] bootstrap | инициализация`).
- `output/.gitkeep` — корень пустой; `черновики/` и прочие подпапки — по факту.

## Шаг 4 — заполнить сопутствующие файлы из ответов интервью

- «О проекте» в `CLAUDE.md` — 1–2 строки про дело + ограничения (валюта/даты, если не дефолтные).
- **Дерево «Архитектура» в `CLAUDE.md`** — вписать созданные подпапки `raw/` (в шаблоне их нет, структура персонализируется здесь).
- «Профиль момента» в `STATE.md`; поле `_Обновлено:_` = сегодня.

## Что НЕ создаётся при bootstrap

- **Роли** — заранее не заводим. Сообщи пользователю, что любую роль (юрист, финансист…) можно создать позже фразой «создай роль <название>» (см. [roles.md](roles.md)).
- Папки в `output/` (включая `черновики/`) и подпапки `raw/` сверх согласованных — по смыслу, когда появится первый файл соответствующего рода (см. правило «Структура растёт по мере надобности» в [CLAUDE.md](../CLAUDE.md)).

После bootstrap — если проект под git — один коммит `bootstrap: инициализация структуры`; без git шаг пропускается.
