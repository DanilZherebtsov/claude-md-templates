# CHANGELOG — business-run-general

Версия шаблона = номер верхней записи. Растёт на +1 при изменении, которое стоит распространять в проекты, инстанцированные из этого шаблона; мелкие правки формулировок версию не двигают. Теги: `[base]` — из общего скелета base-template, `[business]` — собственная специфика класса.

Выровнен с **base-template@v3**.

## v3 — 2026-06-11
`[base]` Выровнен по base v3 (граница СТАНДАРТ/КАСТОМ, ADR-0013): generic ingest/query синхронизированы (метаданные источника, «уверенность из структуры», большой источник); добавлено правило дисциплины «синтез при закрытии единицы работы».

## v2 — 2026-06-11
`[base]` Bootstrap: удалять инструктивную сноску под «О проекте» после заполнения (из base v2).

## v1 — 2026-06-10
Учредительная версия: шаблон стал версионируемой единицей (версия + этот changelog + сверка обновлений на lint). Зафиксировано состояние после переноса рабочего слоя в `output/` и дисциплины синтеза при закрытии. Домен — тип `entities/`, календарь обязательств, роли-линзы, без кода.

> Долг выравнивания: часть base-прозы здесь переписана, а не унаследована verbatim. При «распространить base» выравнивать осознанно (см. [ADR-0011](../../wiki/decisions/adr-0011-versionirovanie-i-rasprostranenie-shablonov.md)).
