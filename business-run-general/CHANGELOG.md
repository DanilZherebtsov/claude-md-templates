# CHANGELOG — business-run-general

Версия шаблона = номер верхней записи. Растёт на +1 при изменении, которое стоит распространять в проекты, инстанцированные из этого шаблона; мелкие правки формулировок версию не двигают. Теги: `[base]` — из общего скелета base-template, `[business]` — собственная специфика класса.

Выровнен с **base-template@v7**.

## v7 — 2026-06-17
`[base][инструкция]` Роли (`methodology/roles.md`): засев — не разовый, роль дообучается на ходу (инджест новых материалов под своим тегом в любой момент, замещённое → `superseded`); при создании человеку — только бизнес-вопросы, срез/куда-писать Claude выводит сам, роль пишет в уместный тип. Источник — боевой AI-education ([ADR-0015](../../wiki/decisions/adr-0015-process-sozdaniya-roli.md), доп. 17.06).

## v6 — 2026-06-16
`[base][инструкция]` Обогащён процесс создания роли (`methodology/roles.md` §«Создание роли»): единый гибкий вход (фраза / файл-описание / + опорные документы); гигиена тегов-префиксов; засев знания из опорных документов в `wiki/discovery/`; **init-проход** ретро-тегирует уже накопленную работу по домену и **завершается синтезом** для человека (что отнесено к роли); подхват новых страниц повторным init. Триггер вызова роли — короткое «работаем в роли <роль>» (путь не называть), синхронно в `CLAUDE.md` §«Роли». Источник — боевой проект AI-education ([ADR-0015](../../wiki/decisions/adr-0015-process-sozdaniya-roli.md)).
