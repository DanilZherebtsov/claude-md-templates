# Ingest — добавление нового знания

Запускается фразами: «обработай этот документ», «запомни это», «добавь в вики», «обнови вики из этой статьи».

Когда человек указывает на новый сырой источник или кладёт файл в `raw/`:

1. **Прочитать источник.** Если он длинный — читать по частям, не пробегать. Сначала обсудить с человеком ключевые выводы и подтвердить намерение перед изменением вики.
2. **Разместить сырой файл.** Сохранить в `raw/<topic>/YYYY-MM-DD-slug.md` (или в подходящую подпапку — для CustDev это `raw/discovery/<client-slug>/`). Сохранить оригинальный текст. Удалить секреты (например, API-ключи) до сохранения. Добавить небольшой метаданный заголовок: URL источника, дата захвата, дата исходной публикации (если известна).
3. **Скомпилировать в вики.** Для каждой затронутой страницы решить:
   - **Тот же тезис, что и у существующей страницы** → объединить. Добавить источник в `sources:`. Обновить затронутые секции точечной правкой, а не полной переписью.
   - **Новая концепция** → создать новую страницу в подходящей type-папке. Назвать по концепции, а не по имени сырого файла. Для discovery — использовать префикс по типу сущности (`client-`, `segment-`, `jtbd-`, `competitor-`). **Выбор типа и формат страницы** — [methodology/page-conventions.md](page-conventions.md).
   - **Противоречит существующему содержанию** → аннотировать конфликт прямо в тексте, обозначить обе стороны и вынести разногласие на разрешение человеком. Не выбирать молча.
4. **Каскад.** Просканировать ту же type-папку и индекс на предмет страниц, чьё содержание материально затронуто. Обновить каждую. Освежить их дату `updated:`.
5. **Обновить `wiki/index.md`** — добавить или обновить записи для каждой затронутой страницы. Формат — [methodology/index-log-format.md](index-log-format.md).
6. **Дописать в `wiki/log.md`**:
   ```
   ## [YYYY-MM-DD] ingest | <название основной страницы>
   - Updated: <название каскадно-обновлённой страницы>
   ```
7. **Закоммитить.** Один коммит на ingest, сообщение: `ingest: <название источника>`.

Важное правило: **при обновлении любого утверждения — перечитайте сырой источник, на который оно ссылается.** Не относитесь к существующему тексту вики как к первоисточнику — именно так заводится дрейф (вики читает собственный вывод и усиливает собственные ошибки).
