# Жизненный цикл task-спек

Спека — это документ конкретной фичи или задачи: проблема, что изменится, реализация, критерии приёмки. Её жизнь — четыре стадии, по подпапкам в `output/specs/`:

```
output/specs/
  pending/     ← Бэклог. Спека написана, в работу не взята.
               Сюда попадают идеи детальнее, чем строчка в STATE.md.
  active/      ← В работе сейчас. Обычно 1-2 файла одновременно.
  accepted/    ← Фича приземлена в код, тесты прошли. Ждёт извлечения ADR.
  rejected/    ← Отказались. Оставляем для истории и поиска.
```

**Имена файлов:** `<feature>-<NNN>-<slug>.md` (например, `auth-012-pkce-flow.md`). Номер сквозной, не сбрасывается между папками — при перемещении файл сохраняет своё имя.

## Переходы между папками

1. **pending → active** — человек или Claude по запросу человека переносит файл, когда задача берётся в работу. STATE.md обновляется одновременно.
2. **active → accepted** — после того как фича приземлена в код и acceptance criteria выполнены. **При этом переходе Claude предлагает выделить ADR**: архитектурное ядро спеки (выбор подхода, отвергнутые альтернативы, последствия) переезжает в `wiki/decisions/adr-NNNN-<slug>.md`. Спека в `accepted/` остаётся как полная история; ADR — короткая canonical-версия для будущих читателей. Двунаправленные ссылки: ADR в `sources:` указывает на спеку, спека в шапке — на ADR.
3. **active → rejected** или **pending → rejected** — отказались от фичи. Опционально создаётся ADR со `status: superseded` (или сразу `rejected`), который объясняет причины отказа — полезно, если позже возникнет соблазн вернуться к идее.

## Правила

- Claude не перемещает файлы между папками автоматически. Только по явной просьбе («перенеси спеку X в accepted») или после подтверждения предложения («задача готова, перенести спеку в accepted и выделить ADR?»).
- Спека в `pending/` или `active/` живёт «свободно» — её можно дописывать, переписывать, разбивать на части. В `accepted/` и `rejected/` спека фактически immutable: содержательные изменения после acceptance идут через ADR (новое решение superseding старое), не через правку спеки задним числом.
- Если спека крупная (десятки страниц, сложная фича) — можно разбить на несколько файлов с общим префиксом (`auth-012-spec.md`, `auth-012-tests.md`, `auth-012-rollout.md`); все они вместе перемещаются между папками.
